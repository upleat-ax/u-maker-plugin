1. TL;DR

Forward Engineering (New Project)

# A. Starting a brand new project → /u-createproject
/u-createproject my-app          # Scaffold Turborepo+Bun monorepo + auto-runs /u-prepare
# ⚠️ After creation: cd my-app && claude  (restart CLI in new folder)

# B. Adding U-MAKER to an existing project → /u-prepare (alias: /u-init)
/u-prepare my-app                 # foldertree + dropzone + analyze/reverse + requirements

# PBGD core pipeline
/u-plan my-app                   # Plan: SRS + IA
/u-wireframe --app my-app --all  # (optional) per-screen HTML wireframes
/u-build my-app                  # Build: Design (ERD/API/Screens/DS) ↔ Dev (FE/BE/DB) orchestrator
/u-gatekeeping my-app            # Gatekeeping: doc scoring + runtime QA (alias: /u-check)
/u-deploy my-app                 # Deploy: interactive target + artifact selection (≥ 98 gate)

# Or run all phases unattended
/u-loop my-app

Reverse Engineering (Existing Codebase)

/u-prepare my-app                 # Initialize .u-maker/ structure first (alias: /u-init)
/u-reverse --app my-app          # Existing code → SSoT reverse engineering

Figma Source Analysis

/u-tools-figma <figma-url>             # Comprehensive Figma analyzer (pages + variants + assets + components + comments + semantics)
# Auto-delegated from /u-prepare, /u-analyze, /u-reverse, /u-build on Figma sources

Natural Language Also Works

"Create SRS for my-app"           # U-MAKER router auto-dispatches to /u-plan
"Generate the ERD for my-app"     # Routed to /u-build --only erd
"Run all tests for my-app"        # Routed to /u-gatekeeping

2. Architecture Overview

U-MAKER accepts both slash commands and natural language, routing them through a skill router and agent dispatch system to produce SSoT documents.

Data Pipeline

data/dropzone/ → data/digest/ → docs/ → output/

3. PBGD 4-Phase Process

PBGD replaces the v3.x PDCA 5-phase model with a tighter 4-phase pipeline: Plan → Build (Design ↔ Dev orchestrated) → Gatekeeping (doc scoring + runtime QA) → Deploy. See CHANGELOG.md for migration details from PDCA.

Phase Definition

PhaseSub-phasesSkillsGate Condition
01 PlanPrepare (foldertree + dropzone + analyze/reverse + requirements) ↔ Plan (SRS + IA + optional wireframes)u-prepare, u-prepare-foldertree, u-analyze, u-reverse, u-plan, u-wireframeSRS + IA Final
02 BuildUI Design ↔ Development orchestrationu-build, u-design, u-devDesign docs Final + code-complete
03 GatekeepingDoc Scoring + Runtime QAu-gatekeeping (aliases u-check, u-qa)Pass ≥ 95
04 DeployCI/CD (interactive target + artifact selection, continuous regeneration)u-deployDeploy-gate ≥ 98
Figma analyzer: /u-tools-figma (owned by u-agent-figma) runs as a cross-phase source analyzer — auto-delegated from /u-prepare, /u-analyze, /u-reverse, and /u-build when Figma URLs or files are present. v4.0.0-alpha.13 ships a reduced extraction path (source.pipeline == "reduced"); downstream consumers treat schema-optional fields as warnings via coverageWarnings[].

Gatekeeper Scorecard

Every gate validation must display the full 11-criteria scorecard (threshold 95, configurable). No abbreviation or omission allowed.

│  GATEKEEPER REPORT — Plan Phase (myapp)                  │
├─────┬──────────────────────┬───────┬────────┬───────────────┤
│  #  │ Criteria             │ Score │ Status │ Findings      │
├─────┼──────────────────────┼───────┼────────┼───────────────┤
│  01 │ Completeness         │   98  │   PASS │ —             │
│  02 │ Accuracy             │   96  │   PASS │ —             │
│  04 │ Traceability         │   88  │   FAIL │ FR-030 orphan │
│  ... (all 11 rows shown) ...                                │
├─────┴──────────────────────┴───────┴────────┴───────────────┤
│ AVG                            │ 95.2  │ PASS   │               │
└─────────────────────────────┴───────┴────────┴───────────────┘
  Attempt: 1/3 · Threshold: 95 · Result: PASS → advance to Design

On FAIL: Improvement items are appended below the scorecard, ordered by severity. After 3 retries: LOOP HALTED — manual intervention required.

7. Gate Criteria

The gatekeeper agent evaluates 11 criteria at each phase transition. The default passing threshold is 95 (configurable). All 11 criteria are always evaluated and displayed.

#CriterionDescription
01CompletenessAll required deliverables exist and contain expected sections
02AccuracyContent correctly reflects source data and requirements
03ConsistencyNo contradictions between documents (e.g., SRS vs. ERD)
04TraceabilityEvery FR → US → FT chain is intact, no orphan IDs
05TOC QualityTable of Contents matches actual document structure
06Content CompositionProper formatting, section depth, prose quality
07Visual AdequacyRequired diagrams and wireframes are present
08Diagram FitnessDiagrams correctly represent described flows and structures
09Mermaid IntegrityAll Mermaid/SVG diagrams render without syntax errors
10JSON SyncCompanion .json files are in sync with .md content
11Cross-ReferenceAll inter-document references (IDs, links) resolve correctly
Threshold is 95 by default. Each criterion is scored 0–100. The average across all 11 must meet or exceed the threshold. Individual criteria that score below threshold are flagged with findings.

Set the document language with language.documents. Supports ko (Korean), en (English), ja (Japanese), zh (Chinese). IDs and code are always in English regardless of this setting.

9. 4-Tier ID Hierarchy

Every requirement is traceable through the hierarchy: USR (User Type) → FR (Functional Requirement) → US (User Story) → FT (Feature / implementation unit). This ensures full traceability from stakeholder needs to delivered code.

FT = Feature (implementation unit). It is not "Functional Test."

11. Install & Prerequisites

Prerequisites

  • Claude Code CLI installed and authenticated
  • Node.js 18+ (for hooks)
  • Git (for /u-tools-git-pr)

Installation

# Install the plugin
claude plugin add upleat-ax/u-maker-plugin

# Restart Claude Code to load the plugin
# Then initialize your project
/u-prepare my-project          # or alias: /u-init my-project

Standalone install scripts

# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/upleat-ax/u-maker-plugin/main/install.sh | bash

# Windows (CMD / cmd.exe)
curl -fsSL --ssl-no-revoke https://raw.githubusercontent.com/upleat-ax/u-maker-plugin/main/install.bat -o install.bat && install.bat && del install.bat

# Windows (PowerShell 5.1+ — `&&` not supported, use `;` and curl.exe)
curl.exe -fsSL --ssl-no-revoke https://raw.githubusercontent.com/upleat-ax/u-maker-plugin/main/install.bat -o install.bat; .\install.bat; del install.bat

Migration from v3.x (PDCA) to v4.x (PBGD)

/u-prepare-foldertree my-project --migrate    # Rename .u-maker/docs/{app}/check/ → gatekeeping/ + re-link graph

Key migration points:

  • Pipeline: Plan → Design → Dev → Check → Ship (5) → Plan → Build → Gatekeeping → Deploy (4).
  • Rename: u-init is now an alias for /u-prepare; actual folder scaffolding is /u-prepare-foldertree.
  • Rename: /u-check is now an alias for /u-gatekeeping.
  • New: /u-deploy (was implicit in "Ship"), /u-tools-figma (reduced path in alpha).
  • Folder: docs/{app}/check/ moves to docs/{app}/gatekeeping/. Use --migrate to rename automatically.

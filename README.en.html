U-MAKER Plugin

PDCA-based SSoT (Single Source of Truth) collaboration orchestrator for Claude Code.
Automates the entire software development lifecycle with 7 specialized agents, 15 skills, and a structured data pipeline.

1. TL;DR

Forward Engineering (New Project)

# 1. Install the plugin
claude plugin add upleat-ax/u-maker-plugin

# 2. Restart Claude Code, then initialize your project
/u-init my-app                # Create .u-maker/ structure + register app

# 3. Collect data → Analyze → Plan
# Drop RFP, meeting notes, AS-IS materials into .u-maker/_dropzone/
/u-plan my-app               # dropzone → digest → SRS + IA + Roadmap

# 4. Design → Implement → Verify
/u-design my-app             # SRS/IA → ERD + API + Screen + Flow + RTM
/u-dev my-app                # ERD/API/Screens → FE + BE + DB code
/u-check my-app              # SRS FT → Test Cases + Test Results

# 5. Ship + Retrospective
/u-ship my-app               # Final verification + iteration log + retrospective

Reverse Engineering (Existing Project)

/u-init my-app                # Initialize project
/u-reverse --app my-app       # Existing code → SSoT reverse engineering

Natural Language Also Works

"Create SRS for my-app"           # U-MAKER router auto-dispatches to /u-plan
"Generate the ERD for my-app"     # Routed to /u-design --only erd
"Run all tests for my-app"        # Routed to /u-check

2. Architecture Overview

U-MAKER accepts both slash commands and natural language, routing them through a skill router and agent dispatch system to produce SSoT documents.

Data Pipeline

data/dropzone/ → data/digest/ → docs/ → output/

Gatekeeper Scorecard

Every gate validation must display the full 11-criteria scorecard (threshold 95, configurable). No abbreviation or omission allowed.

┌─────────────────────────────────────────────────────────────┐
│  GATEKEEPER REPORT — Plan Phase (myapp)                  │
├─────┬──────────────────────┬───────┬────────┬───────────────┤
│  #  │ Criteria             │ Score │ Status │ Findings      │
├─────┼──────────────────────┼───────┼────────┼───────────────┤
│  01 │ Completeness         │   98  │   PASS │ —             │
│  02 │ Accuracy             │   96  │   PASS │ —             │
│  04 │ Traceability         │   88  │   FAIL │ FR-030 orphan │
│  ... (all 11 rows shown) ...                                │
├─────┴──────────────────────┴───────┴────────┴───────────────┤
│ AVG                            │ 95.2  │ PASS   │               │
└─────────────────────────────┴───────┴────────┴───────────────┘
  Attempt: 1/3 · Threshold: 95 · Result: PASS → advance to Design

On FAIL: Improvement items are appended below the scorecard, ordered by severity. After 3 retries: LOOP HALTED — manual intervention required.

  • target: document/item name
  • flags: -i (interactive) | --step | --only X | --cascade | --dry-run | --json

7. Gate Criteria

The gatekeeper agent evaluates 11 criteria at each phase transition. The default passing threshold is 95 (configurable). All 11 criteria are always evaluated and displayed.

#CriterionDescription
01CompletenessAll required deliverables exist and contain expected sections
02AccuracyContent correctly reflects source data and requirements
03ConsistencyNo contradictions between documents (e.g., SRS vs. ERD)
04TraceabilityEvery FR → US → FT chain is intact, no orphan IDs
05TOC QualityTable of Contents matches actual document structure
06Content CompositionProper formatting, section depth, prose quality
07Visual AdequacyRequired diagrams and wireframes are present
08Diagram FitnessDiagrams correctly represent described flows and structures
09Mermaid IntegrityAll Mermaid/SVG diagrams render without syntax errors
10JSON SyncCompanion .json files are in sync with .md content
11Cross-ReferenceAll inter-document references (IDs, links) resolve correctly
Threshold is 95 by default. Each criterion is scored 0–100. The average across all 11 must meet or exceed the threshold. Individual criteria that score below threshold are flagged with findings.

Config Settings

{
  "language": {
    "documents": "en",
    "supported": ["ko", "en", "ja", "zh"]
  }
}

Set the document language with language.documents. Supports ko (Korean), en (English), ja (Japanese), zh (Chinese). IDs and code are always in English regardless of this setting.

9. 4-Tier ID Hierarchy

Every requirement is traceable through the hierarchy: USR (User Type) → FR (Functional Requirement) → US (User Story) → FT (Feature / implementation unit). This ensures full traceability from stakeholder needs to delivered code.

FT = Feature (implementation unit). It is not "Functional Test."

10. Scenario Guide

Project Start (Forward Engineering)

ScenarioCommand
Starting a completely new project/u-init my-project
Have existing RFP / meeting notes/u-init → drop files into .u-maker/_dropzone/ → /u-plan my-app
Migrate from an older u-maker version/u-init my-app --migrate

Project Start (Reverse Engineering)

ScenarioCommand
Full reverse engineering of existing code → SSoT/u-reverse --app my-app
Specify DB schema path/u-reverse --app my-app --db prisma/
Specify API route path/u-reverse --app my-app --api src/app/api/
Specify page component path/u-reverse --app my-app --pages src/app/

Planning (Plan Phase)

ScenarioCommand
Run entire Plan Phase/u-plan my-app
Generate SRS only/u-plan my-app --only srs
Interactive co-creation/u-plan my-app -i
Step-by-step with approval at every step/u-plan my-app --step
Brainstorming session/u-discuss brainstorm "Payment UX"

Design (Design Phase)

ScenarioCommand
Run entire Design Phase/u-design my-app
Generate Screen spec only/u-design my-app --only screens
Generate ERD only/u-design my-app --only erd
Generate all wireframes/u-wireframe --app my-app --all

Development (Dev Phase)

ScenarioCommand
Generate FE + BE + DB code/u-dev my-app
Generate FE only/u-dev my-app --only fe
Generate BE only/u-dev my-app --only be
Generate DB only/u-dev my-app --only db
Prerequisite Cascade: /u-dev auto-invokes /u-design if Design docs are missing; /u-design auto-invokes /u-plan if Plan docs are missing. Running /u-dev alone can trigger the entire Plan → Design → Dev pipeline.

Verification (Check Phase)

ScenarioCommand
Run TC design + test execution/u-check my-app or /u-qa my-app
Review results interactively/u-qa my-app -i

Ship + Retrospective

ScenarioCommand
Final verification + deploy + retrospective/u-ship my-app
Retrospective session/u-discuss retro

Full Automation

ScenarioCommand
Run entire PDCA pipeline unattended/u-loop --app my-app
Run from Plan to Check only/u-loop --app my-app --from plan --to check
Dry run (preview only)/u-loop --app my-app --dry-run

Reporting

ScenarioCommand
Generate HTML output from docs/u-output --app my-app
Generate with SVG diagrams/u-output --app my-app --diagram svg
Daily team report/u-report-daily
Weekly trend report/u-report-weekly
Meeting minutes from notes/u-meeting-report my-app

Collaboration

12. Troubleshooting

SymptomCauseSolution
Plugin commands not recognizedPlugin not loaded or Claude Code not restartedRun claude plugin add upleat-ax/u-maker-plugin and restart Claude Code
/u-plan produces empty SRSNo data in dropzone or digestDrop RFP/meeting notes into .u-maker/_dropzone/ before running /u-plan
Gate always failsThreshold set too high or documents incompleteCheck gatekeeper scorecard findings. Fix flagged criteria. Consider lowering threshold if appropriate
/u-dev generates no codeDesign phase documents missing/u-dev will auto-cascade to /u-design and /u-plan. Ensure dropzone has input data
Dependency cascade not triggering_links.json not initializedRun /u-init to regenerate the dependency graph
/u-loop halts after 3 retriesGate criteria cannot be satisfied automaticallyReview the scorecard, fix the lowest-scoring criteria manually, then resume
Wireframes not renderingScreen spec JSON missing layout fieldRun /u-design --only screens to regenerate screen specs
Wrong document languagelanguage.documents setting mismatchEdit u-maker.config.json and set language.documents to desired locale (ko/en/ja/zh)
/u-reverse misses some filesSource paths not specifiedUse --db, --api, --pages flags to specify exact paths
HTML output missing diagramsDiagram mode not specifiedUse /u-output --diagram svg or --diagram all

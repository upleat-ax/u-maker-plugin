01. What This Plugin Solves

U-MAKER는 Intent-driven 문서 중심 개발(SSoT)을 구현하는 협업 오케스트레이터입니다.

기존 방식U-MAKER v3 방식
코드 먼저, 문서는 나중에문서 먼저, 코드는 문서 기반 생성 Docs-First
기존 프로젝트에 문서가 없음/u-reverse로 소스 코드 → SSoT 자동 역공학
RFP 200페이지를 한 번에 분석 → loss3-Layer 파이프라인으로 chunk 분석 → 정보 손실 없음
"무슨 문서를 만들어야 하지?"/u-plan 한 번이면 SRS+IA+Roadmap 연쇄 생성
요구사항 추적 불가4-Tier ID(USR→FR→US→FT)로 전 구간 추적
SRS 수정 시 ERD/Screen/TC 수동 갱신_links.json 의존성 그래프로 Auto-Cascade
단계 건너뛰기로 품질 저하Phase Gate 자동 검증 후에만 다음 Phase 진행
AI 판단이 블랙박스Assumptions Log로 모든 판단 기록 + 사후 리뷰
설계 산출물 시각화 어려움wireframes/index.html SPA 뷰어 자동 생성
PR/MR 작성이 반복 노동/u-git-pr로 구조화된 PR/MR 자동 생성
Quick Start
  • curl -fsSL https://raw.githubusercontent.com/upleat-ax/u-maker-plugin/main/install.sh | bash
  • Claude Code 재시작 → /u-init my-app → /u-ingest → /u-plan → /u-design → /u-dev → /u-qa → /u-ship

02. Architecture Overview

사용자 입력(slash command 또는 자연어)이 Router를 거쳐 적절한 Agent로 전달되는 구조입니다.

Architecture Overview: User Input → Router → 4 Agents → 13 Engine Skills

03. PDCA 5-Phase Process

Plan → Design → Do → Check → Act 순환. 각 Phase는 Gate 조건을 충족해야 다음으로 진행합니다.

PDCA 5-Phase Lifecycle: 각 Phase Gate를 통과해야 다음 Phase로 전환
Phase산출물담당 AgentGate 조건
01 PlanSRS, IA, Roadmaporchestrator → plannerSRS+IA+Roadmap = Final
02 DesignERD, API, Screen, ScreenFlow, RTM, wireframesplanner → gatekeeperERD+RTM+Screen+API = Final
03 DoCode, UIComponentsbuilderCode complete + build success
04 CheckTestCase, TestReportgatekeeperCritical/Major = 0
05 ActIterationLog, Retrospectiveorchestrator → gatekeeperRetro + archive 완료

04. 3-Layer Data Pipeline

raw data를 한 번에 처리하면 context window 한계로 앞부분이 loss됩니다. 중간 정제 레이어를 두어 해결합니다.

3-Layer Pipeline: Raw → Classified → Deliverables (chunk 분석으로 정보 손실 없음)

05. Agents (4)

4개 전문 에이전트가 Phase별 역할을 분담합니다.

4-Agent Architecture: Orchestrator가 Planner, Builder, Gatekeeper에 작업 분배

u-agent-buildersonnetFE + BE 통합 구현Do
u-agent-gatekeepersonnetGate 검증, TestCase, 일관성 검증, RTMDesign, Check, Act

06. Commands (22)

07. Engine Skills (13)

내부 횡단 엔진. user-invocable이 아니며, command skill이 내부적으로 호출합니다.

Core Orchestrator Engines

  • u-skill-router — Intent 분류, scope 해석
  • u-skill-phase-detector — 문서 status → phase 판정
  • u-skill-dep-engine — _links.json 의존성 그래프
  • u-skill-workflow-runner — Multi-step 실행, rollback
  • u-skill-facilitator — /u-discuss 퍼실리테이션
  • u-skill-backlog — 백로그 CRUD, 스프린트

Domain Planner Engines

  • u-skill-doc-engine — 문서 CRUD, 템플릿 렌더링
  • u-skill-analyzer — Raw 파싱, chunk 분석
  • u-skill-designer — IA/Screen/ERD/API 설계
  • u-skill-estimator — SRS 기반 일정/공수 산정

Execution Builder Engine

  • u-skill-code-engine — 명세 → 코드 생성 (FE+BE)

Validation Gatekeeper Engines

  • u-skill-validator — Gate check, cross-doc 일관성
  • u-skill-test — TC 자동 생성, 실행, 리포트

08. Interaction Modes

Auto default

질문 없이 빠르게 실행. 판단은 Assumptions Log에 기록. maxAssumptions (default: 20) 초과 시 자동 interactive 전환.

Interactive -i

판단 분기에서만 pause. 중요 설계 결정이 있을 때 사용.

Step --step

매 단계에서 pause + 승인. 학습 목적이나 품질 확인 시 사용.

12. Scenario Guide

Forward Engineering (새 프로젝트)

/u-init → _dropzone/에 파일 드롭 → /u-ingest → /u-plan → /u-gate → /u-design → /u-gate → /u-dev → /u-gate → /u-qa → /u-ship

Reverse Engineering (기존 프로젝트)

/u-init → /u-reverse → /u-sync → /u-doc srs --edit → /u-qa

요구사항 변경

/u-add fr "..." → /u-update srs --cascade → /u-status (impact flags 확인)

무인 자동 실행

/u-loop [app] — ingest → plan → design → dev → qa 전체 파이프라인을 한 번에 실행.

Forward Engineering Workflow + Gate Checkpoints + Data Flow
